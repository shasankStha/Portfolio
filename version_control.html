Commands in GIT

  1. git init (Inside of the project folder)
  2. git add (filename) or git add .(To add file)
  3. git commit -m "Comments"(Create commit point)
  4. git config --list
  5. git config --list
  6. git config --list
  7. git config --global user.name""
  8. git config --global user.email""
  9. git checkout -b (branch_name) (Create new branch)
  10. git checkout (branch_name) (Move across branch)
  11. git branch (branch list)
  12. git merge master (Merge changes in the master to branch)
  13. git merge (branch_name) (Merge the changes in the branch to master)

Into Github

  1. git remote add origin <"link">
  2. git push -u origin master (To push)
  3. git pull -u origin master (To pull)

Generating SSH key

In Gitbash terminal,
  1. $pwd (to know location)
  2. $ssh-keygen (to generate key)
Now, open the SSH file from the saved location and copy the key and add it on github.

Repository
A repository is a storage location for software package.

Git vs SVN

GIT SVN
1) Git has Distributed Model. 1) SVN has Centralized Model.
2) In git, every user has their own copy of code on their. 2) In SVN, there is central repository has working copy that makes changes and committed in central repository.
3) In git, we do not required any network to perform git operation. 3) In SVN, we required Network for runs the SVN operation.
4) It is difficult to learn. It has more concepts and commands. 4) SVN is easier to learn compose to git.
5) In git, we create only git directory. 5) In SVN, we create SVN directory in each folder.
6) It doesn't have good UI. 6) It has simple and better UI.
